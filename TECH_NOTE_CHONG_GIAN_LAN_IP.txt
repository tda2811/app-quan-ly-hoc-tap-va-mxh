
HUỚNG DẪN CÀI ĐẶT CHỨC NĂNG CHỐNG GIAN LẬN ĐIỂM DANH (IP NỘI BỘ)
============================================================

1. MỤC TIÊU:
Chỉ cho phép sinh viên điểm danh khi đang kết nối WiFi/Mạng LAN của trường học. Tránh việc nhờ bạn bè ở nhà điểm danh hộ qua App.

2. THƯ VIỆN CẦN THIẾT (Node.js/Express):
- 'request-ip': Để lấy chính xác địa chỉ IP của Client kể cả khi qua Proxy.
- 'ip-range-check': Để kiểm tra IP đó có nằm trong dải IP (Subnet) của trường hay không.

Cài đặt: 
npm install request-ip ip-range-check

3. LOGIC THỰC HIỆN (Tại file routes của Attendance):

// Danh sách dải IP hợp lệ của WiFi trường (Lấy từ IT trường cung cấp)
const SCHOOL_IP_RANGES = [
    '192.168.1.0/24',  // Ví dụ dải WiFi khu A
    '10.0.0.0/16',     // Ví dụ dải Mạng LAN khu B
    '203.162.x.x'      // IP Public của trường (nếu check qua Internet)
];

router.post('/check-in', async (req, res) => {
    const requestIp = require('request-ip');
    const ipRangeCheck = require('ip-range-check');

    // Bước 1: Lấy IP của thiết bị đang gửi request
    const clientIp = requestIp.getClientIp(req); 

    // Bước 2: So sánh với dải IP cho phép
    const isInsideSchool = ipRangeCheck(clientIp, SCHOOL_IP_RANGES);

    if (!isInsideSchool) {
        return res.status(403).json({ 
            success: false, 
            message: 'Bạn phải kết nối WiFi của trường để thực hiện điểm danh!' 
        });
    }

    // Bước 3: Nếu hợp lệ -> Tiến hành lưu vào bảng 'attendances' (lưu cả IP để đối soát)
    // db.query('INSERT INTO attendances (...) VALUES (?, ...)', [..., clientIp]);
});

4. LƯU Ý KHI TRIỂN KHAI:
- Nếu dùng thêm QR Code: SV phải quét mã tại lớp + Check IP mới được coi là hợp lệ (2 lớp bảo mật).
- IPv6: Cần đảm bảo thư viện hỗ trợ cả IPv4 và IPv6 nếu hệ thống mạng trường đời mới.
- Proxy: Luôn dùng 'request-ip' để tránh lấy nhầm IP của Server Load Balancer.
